使用MyBatis遇到的问题

发表于 2016-02-08   |   分类于 Java MyBatis   |  

使用mybits批量插入时的问题

foreach方法

在使用mybits的foreach方法

运行时提示错误信息：

java.util.concurrent.ExecutionException: org.apache.ibatis.binding.BindingException: Parameter ‘__frch_list_0’ not found. Available parameters are [list]

产生原因

我是在传参传入的是List集合，而mybits文档中写道：

注意 你可以传递一个 List 实例或者数组作为参数对象传给 MyBatis。当你这么做的时 候,MyBatis 会自动将它包装在一个 Map 中,用名称在作为键。List 实例将会以“list” 作为键,而数组实例将会以“array”作为键。

解决方法

我将传入的list集合名字改为list也不行，最后是将传入的参数改为map集合，并且参数的名字为map

不重复的向数据库批量添加数据

网站的文章是通过爬虫爬取，所以每次爬取的时候内容可能会重复，对于重复的数据不需要放在数据库中。

解决方法

因为对数据库的接触不深，所以只能按照自己的思维来实现，如果有更好的方法请告诉我，谢谢。

判断两个数据是否一样，我们可以判断其hashcode是否一样。
因为我的需求比较简单，所以我只是根据文章的标题来判断两篇文章是否相同。
每次根据爬取文章的标题生成的hashcode来作为存入数据库时的主键id。

  1. 重写pojo类的equals和hashCode方法（具体实现找谷歌）
  2. 将pojo类的id设置为hashCode的值
  3. 将所有新爬取的数据存放在set集合中
  4. 存入数据库

SQL语句中，insert ignore into…会自动忽略掉主键冲突的数据，这样来实现数据库中数据不重复。
